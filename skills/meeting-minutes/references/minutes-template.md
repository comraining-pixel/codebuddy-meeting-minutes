## 标准会议纪要模板

```markdown
---
title: "会议标题"
category: "meeting"
tags: ["标签1", "标签2", "标签3"]
date: "YYYY-MM-DD"
source: "会议纪要 | 本地PPTX导入 | 音频转写 | 图片OCR | 聊天记录 | 邮件"
status: "active"
participants: ["姓名1(角色)", "姓名2(角色)", "姓名3(角色)"]
related: ["memory/projects-dashboard.md"]
---

# 会议标题

## 基本信息

- **会议时间**：YYYY-MM-DD HH:mm
- **会议类型**：周会 / 专题会 / 项目沟通 / 决策会
- **地点**：线上腾讯会议 / 会议室名称
- **参会人**：姓名1(角色), 姓名2(角色), ...
- **资料来源**：`文件名.pptx` / 音频录音 / 聊天记录 / 图片
- **处理方式**：基于 [解析方式] 整理，仅保留要点、数据和结论，**不保存原始文件**

## 会议摘要

本次会议重点围绕 **主题A、主题B、主题C** 展开。整体看，[一句话概括核心结论和整体态势]。

## 一、章节标题

### 1. 子议题标题

具体内容，包含：
- 数据和事实
- **发言人(角色)** 提出的观点或建议
- 关键结论

### 2. 子议题标题

| 维度 | 数据/内容 | 说明 |
|------|-----------|------|
| 维度1 | 数据 | 说明 |
| 维度2 | 数据 | 说明 |

## 二、章节标题

### 1. 子议题标题

- **观点来源: 姓名(角色)**：具体观点内容
- **观点来源: 姓名(角色)**：具体观点内容

**判断**：基于讨论的总结性判断

## 待办事项

- [ ] 具体待办事项描述 @负责人 截止: YYYY-MM-DD
- [ ] 具体待办事项描述 @负责人 截止: YYYY-MM-DD
- [ ] 具体待办事项描述 @负责人 截止: YYYY-MM-DD

## 关键判断

1. **判断标题**：具体判断内容和理由
2. **判断标题**：具体判断内容和理由
3. **判断标题**：具体判断内容和理由

---

> 📌 备注：本纪要基于 [来源说明] 生成，未保存或归档任何原始文件。
```

## YAML Front Matter 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| title | string | ✅ | 纪要标题，用双引号包裹 |
| category | string | ✅ | 分类: meeting / email / chat / memo |
| tags | string[] | ✅ | 标签数组，反映核心话题 |
| date | string | ✅ | 日期 YYYY-MM-DD |
| source | string | ✅ | 来源描述 |
| status | string | ✅ | 状态: active / archived |
| participants | string[] | ✅ | 参与者列表 "姓名(角色)" |
| related | string[] | ❌ | 关联文档路径 |

## 文件命名规范

- **meetings**: `YYYYMMDD-english-slug.md`
  - 示例: `20260325-wechat-pay-biweekly-meeting.md`
- **decisions**: `slug-YYYYMMDD.md` 或 `slug.md`
  - 示例: `insurance-policy-center-product-plan.md`
- **knowledge**: `slug.md`
  - 示例: `ocr-comparison-guide.md`
- **journal**: 追加到 `YYYY-MM-DD.md`

slug 规则：
- 全小写英文
- 用连字符分隔单词
- 简洁但能表达核心内容
- 不超过 50 个字符
